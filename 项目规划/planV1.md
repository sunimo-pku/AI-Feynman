# AI 费曼学习法 · 项目规划 V1

## 1. 项目目标

**目标用户：初中数学**（初一～初三，全学段覆盖）。

面向该人群打造一款用户定制化、区别于通用大模型的「费曼学习法」**Android App**（Flutter / Dart，平板优先运行）。

- **客户端**：Flutter（Android）
- **服务端**：Python FastAPI（LLM / 语音 / 业务 API）

- **核心差异**：不是通用聊天，而是围绕「讲题 → 追问 → 掌握度更新」的闭环学习
- **商业假设**：家长用户会有更强的付费意愿（可查看学习进度、弱项、精彩讲题回放）

---

## 2. 考核要求（迭代导向）

> 不追求大而全，先做出经用户验证的原型。

| 原则 | 说明 |
|------|------|
| 快速迭代 | 多版迭代，基于反馈迅速出下一版 |
| 找真实用户 | 先有原型 → 用户验证 → 收集反馈 → 再改 |
| 聚焦核心需求 | 不必严格按文档限制；**需求本身最重要** |
| 克制范围 | 功能不做大而全，但**目录结构做全**；实现上先跑通**第十六章 二次根式**的最小闭环 |

## 4. 实时双工交互模式 (边写边讲 & 智能打断)

为了打造极致的陪伴式、启发式讲题体验，系统不采用传统的“单向发送-接收”对讲机模式，而是采用**实时双工流式交互（Continuous Real-time Duplex）**。学生可以一边在手写板书写推导，一边口头讲解，Agent 处于后台持续倾听与分析，并在最合适的时机发出追问。

### 4.1 边写边讲的后台状态机
- **持续采集（Continuous Ingestion）**：
  - **语音端**：前端麦克风长开（取消 60 秒录音限制，支持 5-10 分钟的长段讲题），利用 WebSockets/SSE 将 PCM 音频分片流式上传到后端，由 ASR 引擎在 200ms 内产生流式文本。
  - **笔迹端**：前端手写板实时记录学生书写轨迹的点集、时间戳与边界框（Bounding Box），将其与 ASR 的文本按时间戳建立对齐。
- **Agent 状态决策（Decision Engine）**：
  - 后端常驻一个轻量级的决策组件，根据 ASR 文本的语义完整性、公式推导正确性以及学生的书写停顿（VAD 判定）来决定是否介入。

### 4.2 智能打断机制（Barge-in）
在双工会话中，当 Agent 正在通过 TTS 播放语音或正在流式出字时，若学生有新的动作，系统支持两种维度的**智能打断**：
1. **声音打断（Voice Barge-in）**：
   - 当学生突然开口说话（例如“不对，我这里写错了，应该是...”），前端检测到学生人声（VAD 判定时间 < 100ms），客户端在 100ms 内**强行静音/中止当前播放的 TTS**，清空音频缓冲区，并自动切换为“AI 倾听”状态，将注意力转移到学生最新的讲解中。
2. **笔迹打断（Stroke Barge-in）**：
   - 当 Agent 在发出追问，而学生没有说话但开始在手写板上**落笔书写（产生新的 Stroke 笔迹事件）**，且书写落笔点与追问相关的范围重合或具有极高的书写速度时，前端自动**淡出并停止 TTS 语音**。系统判定学生已理解问题并正在手写板上用笔尖作答，无需等 AI 把话念完。

### 4.3 打断与静默（不打断）的平衡策略
为了让学生能流畅表达，Agent 的介入极其克制，分为以下三种场景：
1. **主动静默（不打断）**：
   - 只要学生处于连续讲解或连续书写状态，即使有小幅度迟疑或短暂的“呃”、“那个”等助词，Agent 依然保持静默不发声，在后台静默收集推导步骤，仅在界面用微弱的波形代表“我在听”。
2. **适时追问（介入打断）**：
   - **错误立即阻断**：学生在推导中写出致命逻辑错误。例如：在化简 \(\sqrt{a \cdot b} = \sqrt{a} \cdot \sqrt{b}\) 时，学生直接写出并讲解，但题目中未给定 \(a \ge 0, b \ge 0\)。Agent 会在其写完这一行并进行口头阐述时，立刻轻微打断：“哎，等等，你这里写下这一步，有没有忽略什么前提条件呀？”。
   - **思维断档提示**：当学生在某一步停下，VAD 判定静音超过 2.5 秒，且手写板无新笔画，说明学生遇到了阻碍。Agent 适时给予间接的启发式提示，例如：“看到你写到这里停下了，你可以想想二次根式里，被开方数应该满足什么范围呢？”。
   - **说毕总结**：当学生口头说出“我讲完了”或静音超过 4 秒且手写板 3 秒无更新，系统触发整体阶段总结。

## 5. 产品形态 · 学生端

### 5.1 双端分离

- **学生端**：讲题、练习、与 Agent 互动
- **家长端**：实时/异步查看学习情况（详见第 5 节）

### 5.2 知识库与题库

- 按**章节 → 知识点 → 题型**切分，系统预置知识库与题库
- 用户可**上传题目**，或由 Agent **生成题目**
- 课本等内容**向量化索引**，Agent 具备检索工具（非整本 PDF 硬塞上下文）
- 选定知识点后，系统**推送对应题目**
- 用户**拍照上传**题目时，Agent 需能**识别所属知识点**

**目录 vs 内容（V1 策略）：**

| 层面 | V1 做法 |
|------|---------|
| **目录** | **做全**：覆盖初中数学完整章节树（初一～初三），用户可浏览全貌、感知产品边界 |
| **内容** | **只填「第十六章 二次根式」**（八年级下册）：16.1～16.3 共 3 节；题目、知识库、讲题流程、掌握度仅在该章落地 |
| **其余章节** | 目录中可见，但无题目/不可进入练习，统一标记为「即将上线」或置灰 |

> 原则：**壳子做全、肉先填一块**——让用户看到完整产品形态，研发资源集中打磨「二次根式」章节的体验。

**V1 上线章节**：八年级下册 · **第十六章 二次根式**（`pep-g8-down-ch16`，含 16.1 / 16.2 / 16.3）

**目录数据文件**：`data/curriculum/pep-junior-math.json`（6 册 · 29 章 · 90 节，含 `id` / `contentStatus` / `v1Launch`）；Flutter 客户端打包为 asset 读取。

### 5.3 学习历史与掌握度

- 每个知识点下记录**练习历史**
- 每次练习后更新该知识点的**掌握等级**
- 掌握度影响后续：**出题难度**、**追问深度**（掌握越好 → 题越难、问越深）

### 5.4 多 Agent 讨论（单个 API 扮演多角色技术方案）

多 Agent 讨论是“AI 费曼”的核心特色。在工程实现上，**不采用并发请求多个大模型 API**（这会导致高昂的延迟、高额的 Token 消耗、难以协调的逻辑冲突以及角色之间上下文的割裂），而是采用**「单大模型上下文多角色剧本生成（Single API, Multi-Role Script Generation）」**的方案，由单个大模型在单次调用中生成完整的讨论场景。

#### 5.4.1 核心人设矩阵
系统预置四个角色（1 个教师 + 3 个虚拟同伴同学）：
- **教师 Agent（把控者）**：温和、引导式。负责控制讨论走向、识别学生是否讲明白、在学生卡住时给与脚手架式的提示、决定讨论的结束时机、并在最后给出结构化总结与变式题。
- **同学 A（基础不牢型 · 小明）**：打破沙锅问到底。专盯定义、前提条件、公理定理的基础漏洞。例如：在二次根式化简时，追问“为什么 \(\sqrt{a^2} = |a|\)？绝对值符号是怎么来的？”。
- **同学 B（计算粗心型 · 大雄）**：容易在乘除法、移项、去括号上犯小脑筋错误，或者挑剔别人（或学生）的细节步骤。例如：“你这里的 \(2 \times 3\) 怎么算成了 \(5\) 啊？算错啦！”的讨论，或者“这步直接跳过了，我是不是可以把 \(\sqrt{12}\) 先化成 \(2\sqrt{3}\) 呀？”。
- **同学 C（总结建议型 · 班长学霸）**：逻辑条理极其清晰。喜欢让学生归纳做这类题的方法、总结易错点，或者给出结构化反馈：“你讲得挺好，但能不能用一句话总结一下，我们怎么判断一个二次根式是不是最简二次根式？”。

#### 5.4.2 技术方案：单大模型多角色剧本生成
后端接收到学生的最新音频（ASR 文本）及笔迹步骤后，向大模型发起单次请求，并在 System Prompt 中将模型定义为**「讨论小组剧本导演」**。

1. **System Prompt 核心结构与上下文组织**：
   - **讨论历史（Context）**：传入之前几轮的对话历史、当前学生的掌握度、题目原文及标准解法。
   - **多角色定义**：定义小明、大雄、班长及老师的具体数学思考水平、说话语调、表情（Emoji）以及性格。
   - **导演规则**：
     - 不要让所有同伴在同一轮都抢着发言，每次最多选择 1-2 个最契合当前学生步骤的角色发言。
     - 同学之间可以根据学生的解法产生“微争论”（例如小明问了一个基础问题，大雄嘲笑他这都不懂，然后大雄自己提了一个计算建议，最后李老师出来主持大局）。
     - 所有发言必须精准对应到学生当前的笔迹步骤 ID（`step_id`）。

2. **结构化 API 契约 (JSON Schema)**：
   为了让前端能完美流式解析、渲染并按序播放多角色对话，API 必须返回强 Schema 约束的结构。即使使用流式 SSE，也应通过 JSON 模式输出。

   ```json
   {
     "discussion_flow": [
       {
         "character": "小明",
         "avatar_type": "student_weak",
         "text": "等等，你在第二步把 \\(\\sqrt{x^2 y}\\) 直接写成了 \\(x\\sqrt{y}\\)，但是题目中写了 \\(y > 0\\)，可没说 \\(x\\) 的正负呀！如果 \\(x\\) 是负数，你这样直接开出来不就错了吗？",
         "voice_tts_text": "等等，你在第二步把 x 平方 y 的算术平方根直接写成了 x 乘根号 y，但是题目中写了 y 大于 0，可没说 x 的正负呀！如果 x 是负数，你这样直接开出来不就错了吗？",
         "highlight_steps": ["step_2"],
         "action_type": "challenge_basic"
       },
       {
         "character": "李老师",
         "avatar_type": "teacher",
         "text": "小明这个提问非常致命！这也是二次根式最容易踩的坑。同学们，你们觉得小明说得对不对？作者同学，你来解释一下为什么你敢直接把 \\(x^2\\) 开出来？",
         "voice_tts_text": "小明这个提问非常致命！这也是二次根式最容易踩的坑。同学们，你们觉得小明说得对不对？作者同学，你来解释一下为什么你敢直接把 x 平方开出来？",
         "highlight_steps": ["step_2"],
         "action_type": "facilitate"
       }
     ],
     "is_finished": false,
     "student_status_update": {
       "current_step_correctness": "partially_correct",
       "feynman_score_delta": 2
     }
   }
   ```

3. **前端流式渲染与播放控制器 (Playback Controller)**：
   由于大模型流式（SSE）吐出的是一个 JSON，前端需要使用**流式 JSON 增量解析库**（或后端把 `discussion_flow` 中的每个 Character 块作为独立的 SSE Event 吐出，例如 `event: character_turn`）：
   - **逐字上屏**：当检测到第一个角色（小明）的内容在流式吐出时，前端对话区的小明气泡立即展现，并开始逐字打字机效果。
   - **TTS 衔接**：在小明的 text 吐出一段并判定其大体语义完整时，前端预先开始向 TTS 引擎请求音频片，或者等小明的流完全结束（`character_turn` 结束），立即播放小明的 ASR 转换后 TTS（使用小明的专属正太音/童音）。
   - **依次发言**：小明播放完后，前端播放器触发 `onComplete` 回调，接着渲染李老师的发言，并播放李老师的 TTS（使用温和的教师男声/女声）。
   - **效果**：学生听到的是极其自然的“连贯讨论会”，而工程上仅需要一次 API 请求与单次流控制，延迟、开销与上下文一致性达到最优。

### 5.5 手写步骤切片与追问高亮机制

当 Agent（如小明或大雄）针对解题过程中的某一部分发出追问时，**手写板（Ink Canvas）上的对应笔迹必须同步产生视觉高亮效果**，让学生能瞬间将“听觉的问句”与“视觉的公式”对齐，消除沟通代差。

#### 5.5.1 手写笔迹与语义步骤的映射 (Ink-to-Step Mapping)
在平板上，学生手写的是一系列由笔划点集构成的 `Stroke`。系统必须将散落的笔划归纳为语义上的“解题步骤”：
1. **空间与时间聚类 (Spatio-Temporal Clustering)**：
   - **断句与换行检测**：当学生手写时，若检测到笔尖离开屏幕时间超过 1.0 秒，且新落笔点与之前的笔迹边界框（Bounding Box）存在明显的垂直向下偏移（Y 轴增加 > 字高），系统判定学生开始书写**新的一行（即新的解题步骤）**。
   - **步骤 ID 绑定 (Step Tagging)**：前端在内部为这一组 Strokes 绑定一个唯一的步骤 ID（例如 `step_1`, `step_2`, `step_3`...）。
2. **实时 OCR 与语义对齐**：
   - 当学生在写完一段并说出讲解词时，前端将带有 Step ID 标记的笔迹数据、以及这些笔迹在 Canvas 上的物理坐标/包围盒发送到后端。
   - 后端轻量级数学 OCR 模型将各步骤的笔迹图像或轨迹转换为 LaTeX 公式（例如 `step_1` 对应 `\sqrt{12} - \sqrt{27}`；`step_2` 对应 `2\sqrt{3} - 3\sqrt{3}`）。
   - 这些公式和 `step_id` 构成的“解题快照”将作为上下文，连同 ASR 文本一起喂给大模型。

#### 5.5.2 追问高亮渲染技术规范
当 API 返回的 `highlight_steps` 包含某个步骤 ID（例如 `["step_2"]`）时，前端手写板进行如下动画高亮：
1. **高亮外框绘制 (Bounding Box Halo)**：
   - 前端在 Canvas 上找出 `step_2` 对应的所有 Strokes 的并集包围盒（Union Bounding Box）。
   - 在该包围盒周围绘制一层浅黄色的半透明霓虹光晕（例如：颜色为 `#FDE047`，带有 `8dp` 的 Blur 圆角外框），并伴有轻微的呼吸动效。
2. **笔迹原色高亮 (Stroke Color Transition)**：
   - 同时，将 `step_2` 的笔迹线条颜色从默认的次要蓝（`#3B82F6`）过渡为知性深蓝（`#1E40AF`）或醒目的智慧湖青（`#0891B2`），粗细微调至 `4.0 dp`。
3. **高亮清除与复位**：
   - 当学生开始新的讲解、或者点击“我懂了”开始下一轮手写时，前端将所有的 `highlight_steps` 动画清除，笔迹恢复为默认颜色，确保画布干净不杂乱。

### 5.6 游戏化激励体系（仿《王者荣耀》荣耀战力与本地排行）

为了让初中生自发、高频、甚至上瘾地进行讲题和“挑错”，系统不采用传统的“学霸小红花”或单调的积分商城，而是**深度像素级致敬《王者荣耀》的战力与排行榜体系**。

初中生的核心心理特点是：**极度在乎同龄人眼中的自我人设（帅、聪明、不服输）**，同时渴望在学校/地区范围内获得某种极具本地辨识度的“吹嘘资本”（例如：济南市历下区第一二次根式终结者）。

#### 5.6.1 “费曼战力值”（Feynman Power Points）
类似于游戏中的“英雄战力值”，每个学生在不同的数学知识维度、章节中，都拥有独立的**“费曼战力值”**（例如：二次根式战力、一次函数战力、几何证明战力）。
1. **战力值构成公式**：
   $$\text{知识点战力} = \text{基础分（正确率）} + \text{讲题表现分（逻辑与流利度）} + \text{挑错表现分（今日悬赏）}$$
2. **战力升降机制**：
   - 当学生成功讲明白一题，战力值增加；如果被虚拟同伴小明或大雄问倒、或者讲解逻辑一团糟，战力可能不加甚至小幅倒扣（提供“战力保分卡”道具，由日常全勤获得，降低挫败感）。
   - **段位进阶**：从“倔强青铜（初学概念）”到“最强王者（金牌讲师）”，段位在个人主页有极其炫酷的徽章展示。

#### 5.6.2 荣耀挑战榜（大榜“大池聚人” + 知识点“微型夺标战”设计）

针对“每一个知识点都设一个排位榜是否可行”的考量，从游戏设计学（Game Design）的角度来看：
- **可行且极具诱惑力的一面**：划分越细，普通学生“拿第一、上榜”的机会就越大。如果只有一个全市大榜，Top 1% 的学霸会永远霸榜，普通孩子会直接放弃；但如果存在“16.1.1 二次根式概念”微型榜，普通孩子只要努力一个周末，就能拿到“历下区第一”，这能极大激发低段位孩子的逆袭快感。
- **致命的陷阱**：**“人数稀释与空服（Ghost Town）”现象**。初中数学共有 90 个大节，细分到微知识点有近 400 个。如果 400 个知识点乘以全国 3000 多个区/县，会产生 **120 万个独立的微型排行榜**。这会导致大部分微型榜里只有 0 或 1 个人，学生轻松拿到“历下区第一”后发现根本没有竞争对手，荣誉感会瞬间贬值、索然无味。

因此，我们采用**「章/节级大榜聚人 + 知识点地盘夺标（抢标）+ 每周焦点擂台」**的精妙混合排位机制：

1. **大榜聚人（池子要大，才有竞争感）**：
   - 排行榜页面（UI 列表）仅设在**「章」**和**「大节」**维度。
   - 例如：*“济南市历下区 · 八年级 · 第十六章 二次根式大榜”*。这保障了榜单内有足够多的活跃学生在实时滚动、互相追赶，榜单含金量十足。

2. **知识点微型夺标战（地盘争夺，称号独占）**：
   - 不为 400 个微知识点做 400 个独立的“列表排位页”，而是将每个微知识点设计成可被**“冠名/争夺”的“武林领地”**。
   - **首席称号（独占性荣誉）**：在某个具体微知识点（如 *16.1.1 二次根式的定义*）下，当前积分第一名的学生将获得该知识点的**独占称号**（例如：*「16.1.1 定义 · 历下区首席辩论员」*）。
   - **领地被夺与复仇（强刺激）**：当学生 B 的积分超过了当前持有人学生 A，称号和光环会立刻转交。
   - 系统向学生 A 推送红字强震动警报：“**警报！济南二中的【张伟】夺走了你在 16.1.1 的‘首席辩论员’称号！点击立刻前往挑错复仇，抢回你的地盘！**”这种“地盘被抢”的强胜负欲刺激，会驱使初中生疯狂进行讲题和纠错。

3. **每周焦点知识点擂台（Traffic Funneling 流量聚焦）**：
   - 平台每周一零点，会选择一个高频易错的微知识点作为**“本周黄金擂台”**（例如：“本周擂台：16.2.2 分母有理化”）。
   - 全省/全市的学生在这一周都会集中在这个“黄金擂台”进行挑错大决战。通过集中流量，既避免了微型榜单的冷清，又创造了每周都有新鲜感、轮流转的微型争霸赛。

通过这种**“大榜看排名，小点抢称号，每周打擂台”**的精细化排位设计，既解决了人数被无限稀释的“空服”死穴，又保障了每个中等生都有机会通过攻克特定知识点，拿到属于自己的“地盘称号”，在班级里自豪地装配和炫耀。

#### 5.6.3 “今日悬赏”：日常挑错排位赛（Daily Bounty Arena）
初中生往往不愿意自己写出完整的步骤（觉得累），但**极度热衷于挑别人的刺、挑同伴的错**。
1. **挑错关卡设计**：
   - 每天 3 道“悬赏易错题”。界面上展示一个名为“大雄”或者“小明”写出的**带有隐蔽漏洞的错误解答笔迹**。
   - **玩法：大家来找茬（数学版）**：
     - **第一阶段：手写画圈**：学生要在手写板上，用红色画笔圈出“出错的那一步笔迹”（通过 5.5 节的 bounding box 包围盒范围判定是否圈对）。
     - **第二阶段：语音纠错（费曼关键）**：学生点击麦克风，开启 ASR。必须用自己的话，把为什么错、以及正确的解法讲出来。
2. **悬赏奖励（双币制）**：
   - **虚拟币：费曼晶石**。用于解锁精美的手写板笔迹皮肤（例如“金色流光笔迹”、“粉色樱花笔迹”）、专属头像框。
   - **战力经验**：直接提升“挑错/纠错战力”，用于冲刺每周一零点的“山东省八年级第一”称号。

#### 5.6.4 实体物理激励：高能极客学习工具局（Geek Study Tools Bureau）

为了将虚拟世界的成就扎实落地，给学生提供强大且长期的自我实现自驱力，系统推出**「高能极客学习工具局」**。学生通过每日悬赏和多 Agent 讲题挑战，获取唯一的代币**“费曼晶石”**，并完全自主地兑换高颜值、专业级的绘图工具和黑科技学习硬件。

- **硬性纯净原则（零充值/零打赏）**：
  - 费曼晶石**绝对不提供任何充值、购买或家长打赏通道**。
  - 每一颗晶石都必须由学生通过严密的逻辑、精准的挑错、流畅的表达一颗一颗“挣”出来。这种 100% 的智力劳动等值性，确立了所兑换实物的极高含金量和尊严感。

- **选品策略（数学核心、黑科技辅助、轻物流高频）**：
  - **专业硬核绘图工具**：如日本百乐（Pilot）或红环（Rotring）全金属防松动专业圆规、多功能平面几何作图钢尺、精密三角板套装。这些是初中数学几何、函数作图的物理“神器”，解决了便宜圆规易松动、画不准的痛点。
  - **黑科技错题整理工具**：便携式无线错题热敏打印机（喵喵机类）。配合学生在“今日悬赏”和“讲题”中积累的错题，晶石可兑换打印纸或打印机，一键打印错题笔迹，贴在物理笔记本上。
  - **自律专注辅助硬件**：重力感应数码番茄时钟。专为初中自习室专注场景设计，物理重力翻转即可开始 25 分钟费曼讲题/写题专注，带有高颜值 LED 律动。
  - **学霸专属错题整理本**：国誉（KOKUYO）定制版“费曼讲题错题本”（采用“左边贴错题、右边梳理多角色追问/费曼步骤”的高效分栏活页）。

- **激励心流**：
  - 学生用自己辛苦攒下的费曼晶石，在 App 内直接、独立、自主地发起兑换。
  - 快递直接寄送至学校或家里。当学生拆开包裹，拿出靠自己“数学智商”赢回来的顶级学习工具并在班级或家里使用时，其独立感、荣誉感和尊贵感将达到峰值，彻底爱上用费曼学习法进行探究和讲题。
