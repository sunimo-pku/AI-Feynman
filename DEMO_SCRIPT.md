# AI 费曼 · Demo 演示提纲

> 每完成一个可演示的功能或逻辑单元，立即在此追加条目（按 `AGENTS.md` 第 0.6 条要求）。
> 演示前请确认：Flutter 客户端已构建到 Android 设备，后端已启动；首次进入 App 会先要求登录或注册。

---

## 1. 登录后进入学生端四 Tab 主界面
- **一句话描述**：打开 App 先进入登录 / 注册页；**学生与家长为独立账号**，必须登录后才能使用（无游客）。学生登录后进入底部导航：**今日 / 课程 / 更多 / 我的**，不再单页堆叠整册目录。
- **演示要点**：
  - 登录页 SegmentedButton 切换「学生账号 / 家长账号」；学生只需账号密码，家长额外填「家长密码」。
  - 注册顺序：先注册学生（如 `xiaoming`）→ 再注册家长（如 `xiaoming_ma`，绑定孩子用户名 `xiaoming`）。
  - 学生登录后默认在「今日」Tab：时段问候 + 推荐小节 + 同伴旁听提示 + 快捷入口；家长登录后**直接进入学习记录页**。
  - 「课程」Tab：仅显示注册年级下的上下册；小节 pill 为左书脊色条 + 纸阴影，无四边描边。
  - 「今日」Tab：「下午好，{昵称}」+ 年级 soft tag；不再显示 KPI 式「掌握度 xx/100」。
  - 「更多」Tab：作业、每日挑战、晶石、榜单、拍照识题、资料编辑。
  - 「我的」Tab：战力与晶石；年级可在资料编辑中修改。
  - 随机点击一个非 16 章节也能进入讲题页，写步骤 / 口述后同样触发多 Agent 追问。
- **预估耗时**：30 秒

## 2. 二次根式讲题闭环（全屏白板 → 头像轨追问）
- **一句话描述**：进入讲题页后整页是白板；顶部窄条可点开看完整题面；右侧小明/大雄/班长/李老师四个圆头像；点「开始讲题」→ 写步骤 →「讲题结束」后看外圈颜色与追问气泡。
- **演示要点**:
  - 全屏手写板（无左侧大矩形对话区）；底部与左下为圆形工具钮（麦克风、讲题结束、撤销、清空等）。
  - 三名同伴头像外圈：绿色=听懂、橙色=没听懂；没听懂时左上角橙色气泡，点开可看追问理由并「听他说」。
  - 李老师头像在轨最下方；有总结时可点开展示。
  - 白板上写几步后点左下「灯泡 · 需要提示」：底部 SnackBar 提示「请看右侧李老师」；李老师头像旁出现内联气泡并自动展开；思考中再点会提示稍候。
  - 讲题结束等待时屏幕中下出现轻量「同伴正在想…」胶囊，不占白板。
  - **只有三名同伴都听懂**（`completed`）后左下才出现「下一题」；有人没听懂时不能跳题，需看右侧追问再讲一轮。
  - 完成后点庆祝圆钮弹出小结 sheet，或直接用「下一题」圆钮切题。
- **预估耗时**：90 秒

## 3. 后端 Mock 多 Agent 追问（第二轮闭环）
- **一句话描述**：在 16.3 讲题页写两步思路 → 点击「提交讲解」→ Flutter 通过 `POST /lecture/submit` 真实请求 FastAPI，后端返回小明 + 李老师两条结构化追问。
- **演示要点**：
  - 演示前先启动后端：`cd main && uvicorn app.main:app --host 0.0.0.0 --port 8001 --reload`，登录与讲题请求都应正常返回。
  - 点击「提交讲解」后按钮进入 loading（旋转图标 + 「正在让同学听讲…」），按钮整体禁用避免重复点击。
  - 后端 ~50ms 返回 200 时，左侧依次出现「系统已收到第 N 轮讲解」「小明追问」「李老师引导」三条气泡，并按 `highlightStepIds` 自动让画布对应 `step_id` 亮起。
  - 关掉 uvicorn 后再点提交，气泡区底部弹出红色错误条「连不上后端（…）。请确认后端已启动…」+「重试」按钮，**手写内容不清空**，重新打开后端后点重试即可继续闭环。
  - 用 `curl -X POST http://127.0.0.1:8001/lecture/submit -H 'Content-Type: application/json' -d '{...}'` 现场展示固定 JSON 契约。
- **预估耗时**：90 秒

## 4. 真实 AI 多 Agent 追问（DeepSeek-V4-Flash 非思考流式）
- **一句话描述**：在讲题页写两步思路 → 点击「开始实时讲题」→ FastAPI 调 DeepSeek-V4-Flash（`thinking.type=disabled`）让模型在一次流式对话里扮演小明/大雄/班长/李老师，实时推送 1-2 条结构化追问气泡。
- **演示要点**：
  - 演示前确认根目录 `.env` 已配置 `DEEPSEEK_API_KEY`，并启动后端 `cd main && uvicorn app.main:app --host 0.0.0.0 --port 8001 --reload`。
  - 按钮文案从「提交讲解」切到「AI 同伴思考中…」，左侧呼吸气泡显示「AI 同伴正在阅读你的步骤、写追问……」，**和第二轮 0.5s 假 loading 明显不同**，让评委直观感到「这次真的在调 LLM」。
  - 后端实时流式返回后，按角色铺出 1-2 条气泡。首条 delta 超过 2 秒会直接报错，不再假装正常。
    - 16.3 实测：小明会精确指出「`\sqrt{12}=2\sqrt{3}` 里为什么开出来是 2 不是 `\sqrt{2}`」，班长引导「合并系数 2 减 3」但不直接给答案 `-\sqrt{3}`。
    - 16.1 实测：班长会要求把规则浓缩成「定义域口诀」，老师会拓展追问「若题目改成 `\sqrt{-x}`，口诀还能直接用吗？条件会变成什么？」。
    - 16.2 实测：小明追问 `\sqrt{a}\cdot\sqrt{b}=\sqrt{ab}` 的前提条件，班长精确给出 `a\ge 0, b\ge 0` 并直接引用 `step_x` 名要求把条件补到那一步前面。
  - 每条气泡都会自动让画布对应 `step_id` 亮起（霓虹光晕 + 笔迹换湖青）。
  - 现场可在终端 `tail -f main/logs/uvicorn.log`，日志里 `source=llm turns=2` 即代表走的是真实 LLM。
  - 用 `curl` 把 `DEEPSEEK_API_KEY` 临时清掉（`DEEPSEEK_API_KEY="" uvicorn ...`）重启后端，再点提交，应看到明确错误而不是假气泡，证明失败不会被 Mock 文案掩盖。
  - 若 LLM 返回非 JSON / 字段非法 / 引用了不存在的 `step_99`，后端校验层会自动剔除或回退首步 `stepId`，前端不会出现"点不亮"或"乱码"。
- **预估耗时**：120 秒

## 5. 学生语义输入驱动 AI 追问（第四轮闭环 · 学生口述 + 每步说明）
- **一句话描述**：在 16.3 讲题页右下角新增「我刚才是这样讲的」+「为每一步补充一句话（可选）」输入区，学生写完两步根号化简后，在讲解框写「我先把 12 拆成 4×3，所以根号 12 可以化成 2 根号 3，再把 27 化成 3 根号 3，最后得到负一根号三」，并在 step_1 / step_2 旁分别写「根号12化成2根号3」「根号27化成3根号3」→ 点提交 → AI 同伴直接**引用学生原话**并质疑其中藏着的前提条件、化简规则、计算符号。
- **演示要点**：
  - 输入区为温和纸张风格卡片，顶部「我刚才是这样讲的」多行输入框（2-4 行），下方按步骤分行展示「`1` 例如：根号 12 化成 2 根号 3」+ 右侧「加公式」收纳 LaTeX 选填，符合 `MOBILE_STYLE.md` 平板优先与 48dp 触控约束。
  - 写新步骤时输入框自动出现；按「清空」画板时步骤说明也一并归零；按「下一题」整片输入区清空；**提交失败 / 重试时手写、讲解、步骤说明都不丢**。
  - 实测真实 LLM 会用中文引号简短照搬学生的关键短语，例如小明追问「你刚才说『根号12变成2根号3』，那12拆成4×3，为什么4能开出来？根号上 $a \cdot b = \sqrt{a}\cdot\sqrt{b}$ 这里需要 $a,b \ge 0$ 吗？」；大雄会盯计算细节追问「你最后说『得到负一根号三』，是 2-3 还是 2+(-3)？写成 $-1\sqrt{3}$ 还是 $-\sqrt{3}$ 更规范？」。
  - 现场可以演示「只手写不补充文字」分支：AI 同伴仍必须根据题面、笔画步骤和已有语音追问；如果 LLM 不可用则直接报错。
  - `curl` 演示：请求体里加上 `"studentSpeechText"`、`"steps[*].plainText"`、`"steps[*].latex"` 三个字段，后端日志稳定打 `source=llm turns=2`，证明真实 LLM 拿到了语义证据。
- **预估耗时**：90 秒

## 6. 学生回答追问 → 多轮上下文闭环（第五轮 · 本地 history + completed 收束）
- **一句话描述**：在 16.3 讲题页提交一次 `\sqrt{12}=2\sqrt{3}` 后，小明追问「为什么 4 可以从根号里出来？」；学生在输入区**直接回答**「因为 12=4×3，4 是完全平方数，规则是 √(ab)=√a·√b，需要 a,b≥0」；点击「回答追问」后，小明先肯定学生答到的部分、再下钻一层（√4·3 vs √(4·3) 的写法、2√3-3√3 的负号）；学生再答清楚后，李老师返回 `status: "completed"`，前端显示「这一题讲清楚了」收束横幅 + 「再讲一遍 / 下一题」按钮。
- **演示要点**：
  - 第一轮提交后，输入区标题自动从「我刚才是这样讲的」**切换**为「回答 X 的追问」（X 取自后端最后一条 AI turn 的 `displayName`）；输入区上方多出一条浅色「**X 刚问：「……」**」复述提示，让学生看清自己要回答的是哪条问题；底部主按钮文案从「再讲一轮」切到「回答追问」；placeholder 也跟着换成「例如：因为 12=4×3，4 是完全平方数，所以可以把 2 提出来……」。
  - 用 `tail -f main/logs/uvicorn.log` 现场展示后端日志格式：`section=… round=2 steps=2 history=2 source=llm status=needs_explanation turns=2`，证明前端确实把上一轮 history 传了进来。
  - 模型对比能力：实测 DeepSeek 第二轮发言会**用引号引用学生第一轮的话**做评价（「你这次说出了『4 是完全平方数』和『a,b≥0』，很好！」），并切换到下一层追问，**不再重复**第一轮原问题；第三轮学生答到位后，李老师收束并精确复述 3 条规则点。
  - completed 状态下：屏幕底部出现绿色完成横幅「这一题讲清楚了。可以点『下一题』继续，也可以点『再讲一遍』自己复盘」；**不自动清空画板**，给学生留时间复盘高亮和笔迹；点「再讲一遍」会清空画板/口述/历史但保留题面，点「下一题」会重置一切回到「提交讲解」初态。
  - 失败重试不丢历史：演示时把后端杀掉，让 round 2 失败，红色错误条出来 + 手写/口述/输入框**全部保留**；重启后端点「重试」，请求体里仍是同一份 history（不重复追加 student 历史），后端日志显示 `history=2` 而不是 `history=4`，体现「构造时临时拼装、成功后才落库」的设计。
  - `curl` 演示三连：分别打 `roundIndex=1` / `roundIndex=2 + history=[student, xiaoming]` / `roundIndex=3 + history=[…, student, xiaoming, student, xiaoming]`，可以现场看出后端在三种 roundIndex 下分别回 `needs_explanation`（多角色追问）→ `needs_explanation`（评价 + 下钻一层）→ `completed`（老师收束）。
  - 失败不伪装：把 `DEEPSEEK_API_KEY` 设成空值重启后端，`/lecture/submit` 应返回 502，前端保留手写和输入内容让学生稍后重试。
- **预估耗时**：120 秒

## 7. 本地学习沉淀 · 首页掌握度变化（第六轮 · 本地 SectionProgress 闭环）
- **一句话描述**：在 16.3 讲题页完成一轮多 Agent 追问，AI 判定 `completed` 后讲题页弹出「本题讲清楚了」小结卡（含本轮小结 + 「本节掌握度 +10 · 当前 10/100」+ 「再讲一遍 / 下一题」），返回首页能看到 16.3 的小节 pill 由「可练习」变为「已完成 1 轮 · 10/100」；关闭 App 重开仍保留。
- **演示要点**：
  - 完成态卡片是**学习反馈基调**，不是游戏化烟花：知性湖青底 + 一行细进度条 + 鼓励但克制的文案「不自动清空画板，可以回看高亮再点『下一题』或『再讲一遍』」。
  - 小结文案优先取最后一条 teacher turn（李老师收束语，常含「请你再用一句话总结：……」），没有就取最后一条 AI turn；都没有再用「本题已完成一轮讲解，建议回看高亮步骤……」的本地文案。**不**会为了总结再调一次 LLM，体感上完成态卡片在 completed 落地后**立刻**出现。
  - 讲题页 AppBar 右上角徽标在 completed 后从「理解中 / 本轮第 N 次提交」**自动切**到「本节 X/100 · 已完成 N 轮」，同步反映跨题持续的学习收益；这是 `ProgressRepository`（`ChangeNotifier`）通知首页 / AppBar 同时刷新的效果。
  - 返回首页：16.3 小节 pill 的圆形图标从 `play_circle_outline` 切到 `check_circle`，右侧徽标从「可练习」切到「已完成 1 轮 · 10/100」；其他未练习小节保持「3 道题 · 可练习」。
  - 重启 App：杀掉进程再开，首页直接显示「已完成 1 轮 · 10/100」，证明数据真的写进了 `shared_preferences`（key：`ai_feynman.section_progress.v1`）。
  - 多次完成累加：在 16.3 再完成 9 轮，掌握度 `+10/+10/...` 到 90，第 10 轮后封顶 100（不溢出），小结卡下方提示「本节掌握度 已封顶 · 当前 100/100」。`masteryDelta <= 0` 但仍 completed 时至少加 8 分，保证学生**完成一题永远拿到正反馈**。
  - 跨小节独立：在 16.1 / 16.2 / 16.3 三节分别完成，首页能看到各自独立的「已完成 N 轮 · X/100」徽标。
  - 失败容错：手动清空 `shared_preferences` 后重开 App（或第一次安装），首页 16.3 自动回落到「可练习」，**不**显示 `null` / `--`，不影响课程目录展示；后端临时不可用也不影响进度展示（进度是纯本地的）。
- **预估耗时**：90 秒

## 8. 多章节 Mock 题切换
- **一句话描述**:同一讲题页根据章节展示不同的二次根式 Mock 题（16.1 取值范围 / 16.2 乘除 / 16.3 加减）。
- **演示要点**：返回首页 → 进入 16.1 看到「`\sqrt{2x-6}` 有意义时 $x$ 的取值范围」；进入 16.2 看到「`\sqrt{12} \cdot \sqrt{3}`」；每道题 Mock 对话都围绕该节核心知识点（被开方数非负 / `\sqrt{a}\cdot\sqrt{b}=\sqrt{ab}` 的前提 / 同类二次根式）。
- **预估耗时**：60 秒

## 9. 同一小节内多题轮换 + 难度标签（第七轮 · 本地小题库 + 下一题循环）
- **一句话描述**：同一小节内置 3 道题（基础 / 巩固 / 挑战），讲题页 AppBar 与题面卡片清楚标出「第 N / 3 题」+ 难度 chip + 1-3 个知识标签 chip；点「下一题」真正切到本节下一道题、第 3 题循环回第 1 题；点「再讲一遍」保持当前题不变；切题后后端请求体里 `questionId` / `questionPrompt` 也跟着同步。
- **演示要点**：
  - **首页提示题量**：进首页可看到 90 个小节均带「3 道题 · 可练习」徽标；所有章节完成过后都会切到「已完成 N 轮 · X/100」覆盖题量徽标，方便家长一眼看出"哪节练过、哪节还没碰"。
  - **题面卡片完整信息**：进入 16.3，AppBar 标题显示「16.3 二次根式的加减 · 第 1 / 3 题」，题面卡顶行显示「16.3 二次根式的加减 · 第 1 / 3 题」，下方一行知性湖青「基础」chip + 「最简二次根式」「同类二次根式」两个标签 chip，最后才是 LaTeX 题面 `\sqrt{12}-\sqrt{27}` + 提示。
  - **难度 chip 风格不同**：基础（湖青）/ 巩固（深蓝）/ 挑战（警示红，但仅做点缀，不抢主视觉）；都是 8dp 圆角、12sx 中文字号，符合 `MOBILE_STYLE.md` 的「小而清楚、远离游戏化徽章」要求。
  - **下一题真切题不是重置**：在 16.3 第 1 题写两步、提交、AI 追问后点「下一题」，AppBar 标题切到「第 2 / 3 题」，题面变成 `2\sqrt{8}+\sqrt{18}`、难度 chip 切到「巩固」、标签 chip 切到「化简」「合并同类项」；老师 intro 气泡复述新题号「这是本节的第 2 / 3 题（巩固）」，画板 / 口述 / 步骤说明 / `history` / `turns` 全部清空，提交按钮回到「提交讲解」初态。
  - **第 3 题点下一题循环回第 1**：再点两次「下一题」分别到第 3 / 3 题（挑战 · `\sqrt{45}-2\sqrt{20}+\sqrt{5}`）和第 1 / 3 题（基础），证明索引是 modulo 而不是 max 卡死，也不会抛异常。
  - **「再讲一遍」保留同题**：在 completed 收束态点「再讲一遍」，AppBar 题号 / 题面 / 难度 / 标签 chip 全保持当前题，但画板 / `history` 全清；intro 气泡之后追加一条「好，再讲一遍……」system 气泡，提示学生这次试着用更精炼的总结。
  - **后端契约同步**：`tail -f main/logs/uvicorn.log` 现场可看到切题前后 `question=q-s16-3-001` → `question=q-s16-3-002` 真实变化，证明 `questionId` 随题轮换；LLM 追问围绕新题面进行。若 `DEEPSEEK_API_KEY` 缺失，讲题应直接报错。
  - **完成新题继续累计**：在第 2 题完成一轮 → 16.3 小节掌握度从 10/100 涨到 20/100；返回首页确认；证明"题级切换"不影响"小节级累计"（第六轮的 `SectionProgress` 没动）。
  - **平板防溢出**：竖屏模拟器下 chip 行会自动 `Wrap` 换行，**不**挤压手写板主区域；横屏 12 寸平板下三个 chip 整齐排在一行。
- **预估耗时**：120 秒

## 10. 本地讲题回顾与再讲这题（第八轮 · 本地 review record + 错因卡片）
- **一句话描述**：在 16.3 第 2 题完成一轮多 Agent 追问、AI 判定 `completed` 之后，返回首页 16.3 小节 pill 旁的「回顾」入口由灰转湖青，点进去能看到刚刚完成的题目卡片（题面 + 难度 / 标签 chip + 完成时间 + 本题总结 + AI 追问摘要 + 下次要注意点），点「再讲这题」直接回到讲题页**定位到 16.3 第 2 题**而不是默认第 1 题；重启 App 后回顾记录仍在。
- **演示要点**：
  - **回顾入口随状态联动**：首页加载后，每个可练习小节 pill 旁会出现一个小型「回顾」按钮（图标 + 文字）；没有任何记录时是浅灰描边、副文本提示「还没完成过题」（仍可点，进去看空状态）；有过 ≥1 条记录后整体切到湖青高亮，告诉学生「这里有东西可看」。未上线章节**不**展示回顾入口。
  - **空状态文案温和不焦虑**：第一次进入回顾页时，居中显示书本 icon + 「完成一题后，这里会出现你的讲题小结和 AI 追问。」+「回到上一页，进入这一节的题目讲解，完成后回来看回顾。」，**没有**任何「您还没有任何记录！」的红色感叹号设计。
  - **完成一题真的落地一条 review**：在 16.3 第 2 题（`2\sqrt{8}+\sqrt{18}`）走一轮 AI 追问 → AI 判定 `completed` → 回顾页立刻出现一条新卡片，顶行「刚刚完成」+「本题总结」段保留 LaTeX 渲染（`\sqrt{8}=2\sqrt{2}` 不会变乱码）+「AI 追问摘要」段抠出最后 1-3 条 AI turn 的关键短语（每条 ≤80 字，超长自动加省略号）+「下次要注意」段按题目标签 `["化简", "合并同类项"]` 自动生成「先化成最简二次根式，再合并同类项系数。」一条规则。
  - **多题完成后倒序展示**：连续完成 16.3 第 2 → 第 3 → 第 1 题（用「下一题」轮换），回顾页按 `completedAt` **倒序**展示：最上面是刚才的第 1 题、下面是第 3 题、再下面是第 2 题。每条卡片自带独立的「再讲这题」按钮。
  - **「再讲这题」精准定位**：在回顾页点 16.3 第 2 题卡片的「再讲这题」 → 进入讲题页 AppBar 直接显示「16.3 二次根式的加减 · 第 2 / 3 题」，题面是 `2\sqrt{8}+\sqrt{18}`，**不是**默认的第 1 题；画板 / 口述 / `history` / `_round` 都是初态，progress / review 历史不被擦除。
  - **极端情况兜底**：如果某天题库 `questionId` 改名了，老 review 记录里的 `questionId` 找不到对应题目 → LecturePage 内部自动回落到该小节第 1 题，不抛异常、不让学生看到 404。可以现场用 `flutter run --dart-define=DEBUG_FAKE_QID=q-not-real` 之类的方式模拟（或者直接改 mock 题库 questionId 重启演示）。
  - **重启 App 仍在**：杀掉进程再开 App，回顾页内容完全保留 —— 证明数据真的写进了 `shared_preferences`（key：`ai_feynman.lecture_reviews.v1`）。
  - **容量上限观感**：连续完成 30+ 题后，全局只保留最近 30 条；单小节回顾页最多展示最近 10 条。最旧的记录自动淘汰，**不**让 prefs 文件无限增长拖垮启动速度。
  - **错因卡片是规则生成的**：现场强调「**没有**为了生成『下次要注意』再调一次 LLM」—— 这是按题目 tags 跑的本地规则，含「非负条件」「合并同类项」「负号」等 5 条硬规则 + 兜底文案，零网络成本，completed 后**立刻**出现，体感与小结卡同步。
  - **失败容错**：手动把 prefs 里 `ai_feynman.lecture_reviews.v1` 改成乱码 → 重启 App → 回顾页不崩、显示空状态；按下一题再完成一道 → 新记录正常写入。
- **预估耗时**：120 秒

## 11. 手动语音讲题 · 开始讲题 → 讲题结束 → 流式追问 → TTS（第九轮 · 学生端 MVP 总收口）
- **一句话描述**：在 16.3 第 1 题点「开始讲题」，孩子一边在右侧白板写 `\sqrt{12}=2\sqrt{3}`、一边口头讲解「我先把 12 拆成 4×3，4 是完全平方数…」；讲完后像微信群聊发语音一样点「讲题结束」，左侧逐字流式出现小明追问气泡、对应白板步骤亮起霓虹光晕，紧接着 TTS 真人腔朗读追问。
- **演示要点**：
  - **手动语音段落**：右侧白板下方的「实时音频面板」从「开始讲题」→「正在听你讲...」→「讲题结束」→「AI 正在想问题...」→「AI 同伴正在说话」切换；停顿、犹豫、换气都不会自动触发追问。
  - **底层数据流可观察**：`tail -f main/logs/uvicorn.log` 现场展示 `[live-session] session_start … audio_chunk seq=N … asr_segment text="…" … thinking … agent_turn_start … agent_turn_delta … agent_turn_done … round_done`，每一条都能在前端找到对应反应。
  - **流式气泡逐字增长**：后端把 LLM 输出按 ~20 字一段切成 `agent_turn_delta` 推给前端，气泡随 delta 增长，**不再是「呼吸 10s 后整段一次出现」**。每条 delta 间隔 40ms，前端 setState 平稳不抖。
  - **白板步骤随气泡高亮**：`agent_turn_start.highlightStepIds` 命中谁，对应笔迹立刻换湖青 + 浅黄霓虹光晕；多 turn 时 highlight 切换跟随当前发言角色。
  - **TTS 同步播放**：第一条 `agent_turn_done` 触发 `/tts` 合成 mp3 → `audioplayers` 播放；嗓音是 `zh_female_qingchezizi_moon_bigtts`（火山豆包默认女声）；同一轮内多条 turn 顺序排队播放。
  - **无自动打断**：TTS 正在说时，学生开口或落笔都不会发送 `student_interrupt`，AI 会把当前追问播完，避免误触发造成体验割裂。
  - **手动「讲题结束」**：孩子讲完后点面板里的「讲题结束」才触发 `pause_detected`；前端同时停止本段录音，AI 追问结束后回到「开始讲题」，下一轮再手动录一段。
  - **白板 snapshot 实时同步**：每次新增 / 撤销 / 清空白板后 480ms debounce 触发一次 `ink_snapshot` 上送；后端 prompt 拼装时拿到的是**最新**的 step 列表 + 笔画数 + boundingBox，不是上一次提交时的快照。
  - **完成后仍写本地进度 / 回顾**：当后端推 `round_done(status=completed, masteryDelta=1)`，前端复用第六/八轮逻辑：弹小结卡 + 写 `SectionProgress` + 写 `LectureReviewRecord`，返回首页 16.3 小节 pill 显示「已完成 1 轮 · 10/100」，回顾页多一条新卡片。
  - **故障兜底闭环**：
    - 把后端 uvicorn 杀掉 → 面板红色「连接断开，白板还在，可以重新开始」+ 「重新连接 / 用文字提交」两个按钮；点「用文字提交」回到第四轮的文字输入区 + 第五轮 `/lecture/submit` 多轮闭环。
    - 系统设置里禁掉麦克风权限重开 → 面板切到「麦克风权限被拒绝」，副文本提示「请到系统设置 → 应用权限里允许『麦克风』后再试」+ 兜底「用文字提交」可点。
    - `DEEPSEEK_API_KEY` 设成空字符串 → 后端 `pause_detected` 后发送 WebSocket `error`，前端进入失败态并显示原因，不出现假追问气泡。
  - **不展示完整 ASR 转写**：默认模式下白板 / 面板 / 对话区都**不**展示 ASR 转写文本（避免学生纠结识别错别字）；只有 `--dart-define=LIVE_FALLBACK_TEXTINPUT=1` 启动时才会把 ASR 片段累加到底部「我刚才是这样讲的」文字框里供 demo 演示者校对。
  - **后端实时性指标可对账**：实时追问使用 DeepSeek-V4-Flash 流式非思考模式；首条 delta 超过 2 秒直接报错，不再等待慢模型整段返回。
- **预估耗时**：180 秒
