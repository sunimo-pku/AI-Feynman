# AI 费曼 · Demo 演示提纲

> 每完成一个可演示的功能或逻辑单元，立即在此追加条目（按 `AGENTS.md` 第 0.6 条要求）。
> 演示前请确认：Flutter 客户端已构建到 Android 设备，后端可选启动（首页右上角 API 标识会变绿）。

---

## 1. 课程目录全貌展示
- **一句话描述**：进入 App 首页，展示「人教版 · 初中数学」6 册 29 章 90 节完整目录，验证「壳子做全、肉先填一块」。
- **演示要点**：
  - 顶部 Hero 卡片指明「V1 仅开放八年级下册 · 第十六章 二次根式」，附带 16.1 / 16.2 / 16.3 三个绿色徽标。
  - 默认展开八年级下册，可练习章节以知性蓝高亮、带「可练习」标签；其他章节灰色 + 锁形图标 + 「即将上线」。
  - 点击未开放章节弹出温和提示，不进入练习页。
- **预估耗时**：30 秒

## 2. 二次根式讲题闭环（手写 → 多 Agent 追问）
- **一句话描述**：点击「16.3 二次根式的加减」进入讲题页，写出 `\sqrt{12}-\sqrt{27}` 的两行化简过程并提交，左侧小明 + 李老师按角色追问。
- **演示要点**:
  - 横屏平板呈现左 40% 多 Agent 对话区 + 右 60% 手写板的双栏布局。
  - 右侧手写板带横线辅助、`RepaintBoundary` 隔离重绘；支持「撤销」「清空」「提交讲解」。
  - 每停笔 1 秒以上自动切到下一 `step_id`，右上角实时显示「已识别 N 步」。
  - 点击「提交讲解」后约 0.6s 出现「正在让同学听讲…」呼吸气泡，紧接着出现小明的疑问 + 李老师的引导。
  - 气泡里的「看这一步」按钮可触发画布对应 `step_id` 的浅黄霓虹光晕 + 笔迹换色（湖青）效果。
  - AppBar 右上角徽标随轮次切换为「已完成 N 轮」。
- **预估耗时**：90 秒

## 3. 后端 Mock 多 Agent 追问（第二轮闭环）
- **一句话描述**：在 16.3 讲题页写两步思路 → 点击「提交讲解」→ Flutter 通过 `POST /lecture/submit` 真实请求 FastAPI，后端返回小明 + 李老师两条结构化追问。
- **演示要点**：
  - 演示前先启动后端：`cd main && uvicorn app.main:app --host 0.0.0.0 --port 8001 --reload`，首页右上角徽标变绿表示「API 已连接」。
  - 点击「提交讲解」后按钮进入 loading（旋转图标 + 「正在让同学听讲…」），按钮整体禁用避免重复点击。
  - 后端 ~50ms 返回 200 时，左侧依次出现「系统已收到第 N 轮讲解」「小明追问」「李老师引导」三条气泡，并按 `highlightStepIds` 自动让画布对应 `step_id` 亮起。
  - 关掉 uvicorn 后再点提交，气泡区底部弹出红色错误条「连不上后端（…）。请确认后端已启动…」+「重试」按钮，**手写内容不清空**，重新打开后端后点重试即可继续闭环。
  - 用 `curl -X POST http://127.0.0.1:8001/lecture/submit -H 'Content-Type: application/json' -d '{...}'` 现场展示固定 JSON 契约。
- **预估耗时**：90 秒

## 4. 真实 AI 多 Agent 追问（第三轮闭环 · Kimi K2.6 关思考）
- **一句话描述**：在 16.3 讲题页写两步 `\sqrt{12}=2\sqrt{3}`、`\sqrt{27}=3\sqrt{3}` → 点击「提交讲解」→ FastAPI 调 `kimi-k2.6`（`thinking.type=disabled`）让旗舰模型在一次对话里同时扮演小明/大雄/班长/李老师，~10-15 秒后给出 1-2 条结构化追问气泡，每条都精准命中学生写的那两个 `step_id`。
- **演示要点**：
  - 演示前确认根目录 `.env` 已配置 `KIMI_API_KEY`，并启动后端 `cd main && uvicorn app.main:app --host 0.0.0.0 --port 8001 --reload`。
  - 按钮文案从「提交讲解」切到「AI 同伴思考中…」，左侧呼吸气泡显示「AI 同伴正在阅读你的步骤、写追问……」，**和第二轮 0.5s 假 loading 明显不同**，让评委直观感到「这次真的在调 LLM」。
  - 后端 ~10-15s 返回后，按角色铺出 1-2 条气泡。**重点是质量而不是速度**：
    - 16.3 实测：小明会精确指出「`\sqrt{12}=2\sqrt{3}` 里为什么开出来是 2 不是 `\sqrt{2}`」，班长引导「合并系数 2 减 3」但不直接给答案 `-\sqrt{3}`。
    - 16.1 实测：班长会要求把规则浓缩成「定义域口诀」，老师会拓展追问「若题目改成 `\sqrt{-x}`，口诀还能直接用吗？条件会变成什么？」。
    - 16.2 实测：小明追问 `\sqrt{a}\cdot\sqrt{b}=\sqrt{ab}` 的前提条件，班长精确给出 `a\ge 0, b\ge 0` 并直接引用 `step_x` 名要求把条件补到那一步前面。
  - 每条气泡都会自动让画布对应 `step_id` 亮起（霓虹光晕 + 笔迹换湖青）。
  - 现场可在终端 `tail -f main/logs/app_*.log`，日志里 `source=llm turns=2` 即代表走的是真实 LLM；`source=fallback` 即走第二轮 Mock 兜底。
  - 用 `curl` 把 `KIMI_API_KEY` 临时清掉（`KIMI_API_KEY="" uvicorn ...`）重启后端，再点提交，气泡照常出现 ——**Demo 链路永不中断**，说明 fallback 已就绪。
  - 若 LLM 返回非 JSON / 字段非法 / 引用了不存在的 `step_99`，后端校验层会自动剔除或回退首步 `stepId`，前端不会出现"点不亮"或"乱码"。
- **预估耗时**：120 秒

## 5. 学生语义输入驱动 AI 追问（第四轮闭环 · 学生口述 + 每步说明）
- **一句话描述**：在 16.3 讲题页右下角新增「我刚才是这样讲的」+「为每一步补充一句话（可选）」输入区，学生写完两步根号化简后，在讲解框写「我先把 12 拆成 4×3，所以根号 12 可以化成 2 根号 3，再把 27 化成 3 根号 3，最后得到负一根号三」，并在 step_1 / step_2 旁分别写「根号12化成2根号3」「根号27化成3根号3」→ 点提交 → AI 同伴直接**引用学生原话**并质疑其中藏着的前提条件、化简规则、计算符号。
- **演示要点**：
  - 输入区为温和纸张风格卡片，顶部「我刚才是这样讲的」多行输入框（2-4 行），下方按步骤分行展示「`1` 例如：根号 12 化成 2 根号 3」+ 右侧「加公式」收纳 LaTeX 选填，符合 `MOBILE_STYLE.md` 平板优先与 48dp 触控约束。
  - 写新步骤时输入框自动出现；按「清空」画板时步骤说明也一并归零；按「下一题」整片输入区清空；**提交失败 / 重试时手写、讲解、步骤说明都不丢**。
  - 实测真实 LLM（K2.6 关思考）会用中文引号简短照搬学生的关键短语，例如小明追问「你刚才说『根号12变成2根号3』，那12拆成4×3，为什么4能开出来？根号上 $a \cdot b = \sqrt{a}\cdot\sqrt{b}$ 这里需要 $a,b \ge 0$ 吗？」；大雄会盯计算细节追问「你最后说『得到负一根号三』，是 2-3 还是 2+(-3)？写成 $-1\sqrt{3}$ 还是 $-\sqrt{3}$ 更规范？」。
  - 现场可以演示「全空提交」分支：学生只手写不补充任何文字，AI 同伴会自动回退到「泛泛追问本节核心知识点」，不会因为新字段而崩。
  - `curl` 演示：请求体里加上 `"studentSpeechText"`、`"steps[*].plainText"`、`"steps[*].latex"` 三个字段，后端日志稳定打 `source=llm turns=2`，证明真实 LLM 拿到了语义证据。
- **预估耗时**：90 秒

## 6. 学生回答追问 → 多轮上下文闭环（第五轮 · 本地 history + completed 收束）
- **一句话描述**：在 16.3 讲题页提交一次 `\sqrt{12}=2\sqrt{3}` 后，小明追问「为什么 4 可以从根号里出来？」；学生在输入区**直接回答**「因为 12=4×3，4 是完全平方数，规则是 √(ab)=√a·√b，需要 a,b≥0」；点击「回答追问」后，小明先肯定学生答到的部分、再下钻一层（√4·3 vs √(4·3) 的写法、2√3-3√3 的负号）；学生再答清楚后，李老师返回 `status: "completed"`，前端显示「这一题讲清楚了」收束横幅 + 「再讲一遍 / 下一题」按钮。
- **演示要点**：
  - 第一轮提交后，输入区标题自动从「我刚才是这样讲的」**切换**为「回答 X 的追问」（X 取自后端最后一条 AI turn 的 `displayName`）；输入区上方多出一条浅色「**X 刚问：「……」**」复述提示，让学生看清自己要回答的是哪条问题；底部主按钮文案从「再讲一轮」切到「回答追问」；placeholder 也跟着换成「例如：因为 12=4×3，4 是完全平方数，所以可以把 2 提出来……」。
  - 用 `tail -f main/logs/uvicorn.log` 现场展示后端日志格式：`section=… round=2 steps=2 history=2 source=llm status=needs_explanation turns=2`，证明前端确实把上一轮 history 传了进来。
  - 模型对比能力：实测 K2.6 第二轮发言会**用引号引用学生第一轮的话**做评价（「你这次说出了『4 是完全平方数』和『a,b≥0』，很好！」），并切换到下一层追问，**不再重复**第一轮原问题；第三轮学生答到位后，李老师收束并精确复述 3 条规则点。
  - completed 状态下：屏幕底部出现绿色完成横幅「这一题讲清楚了。可以点『下一题』继续，也可以点『再讲一遍』自己复盘」；**不自动清空画板**，给学生留时间复盘高亮和笔迹；点「再讲一遍」会清空画板/口述/历史但保留题面，点「下一题」会重置一切回到「提交讲解」初态。
  - 失败重试不丢历史：演示时把后端杀掉，让 round 2 失败，红色错误条出来 + 手写/口述/输入框**全部保留**；重启后端点「重试」，请求体里仍是同一份 history（不重复追加 student 历史），后端日志显示 `history=2` 而不是 `history=4`，体现「构造时临时拼装、成功后才落库」的设计。
  - `curl` 演示三连：分别打 `roundIndex=1` / `roundIndex=2 + history=[student, xiaoming]` / `roundIndex=3 + history=[…, student, xiaoming, student, xiaoming]`，可以现场看出后端在三种 roundIndex 下分别回 `needs_explanation`（多角色追问）→ `needs_explanation`（评价 + 下钻一层）→ `completed`（老师收束）。
  - fallback 兜底也跟着多轮：把 `KIMI_API_KEY` 设成 `your_kimi_api_key_here` 重启后端，第 1 轮 fallback 出 2 条固定追问、第 2 轮 fallback 直接收到「李老师：嗯，从你刚才补的解释看，思路已经顺多了……」+ `status: completed`，**不会两次都看到一模一样的 Mock 追问**。
- **预估耗时**：120 秒

## 7. 本地学习沉淀 · 首页掌握度变化（第六轮 · 本地 SectionProgress 闭环）
- **一句话描述**：在 16.3 讲题页完成一轮多 Agent 追问，AI 判定 `completed` 后讲题页弹出「本题讲清楚了」小结卡（含本轮小结 + 「本节掌握度 +10 · 当前 10/100」+ 「再讲一遍 / 下一题」），返回首页能看到 16.3 的小节 pill 由「可练习」变为「已完成 1 轮 · 10/100」；关闭 App 重开仍保留。
- **演示要点**：
  - 完成态卡片是**学习反馈基调**，不是游戏化烟花：知性湖青底 + 一行细进度条 + 鼓励但克制的文案「不自动清空画板，可以回看高亮再点『下一题』或『再讲一遍』」。
  - 小结文案优先取最后一条 teacher turn（李老师收束语，常含「请你再用一句话总结：……」），没有就取最后一条 AI turn；都没有再用「本题已完成一轮讲解，建议回看高亮步骤……」的兜底。**不**会为了总结再调一次 LLM，体感上完成态卡片在 completed 落地后**立刻**出现，不再多花 5-15s 等 Kimi。
  - 讲题页 AppBar 右上角徽标在 completed 后从「理解中 / 本轮第 N 次提交」**自动切**到「本节 X/100 · 已完成 N 轮」，同步反映跨题持续的学习收益；这是 `ProgressRepository`（`ChangeNotifier`）通知首页 / AppBar 同时刷新的效果。
  - 返回首页：16.3 小节 pill 的圆形图标从 `play_circle_outline` 切到 `check_circle`，右侧徽标从「可练习」切到「已完成 1 轮 · 10/100」；其他未练习小节保持「可练习」；未上线章节仍是「即将上线 + 锁形」**不**显示进度。
  - 重启 App：杀掉进程再开，首页直接显示「已完成 1 轮 · 10/100」，证明数据真的写进了 `shared_preferences`（key：`ai_feynman.section_progress.v1`）。
  - 多次完成累加：在 16.3 再完成 9 轮，掌握度 `+10/+10/...` 到 90，第 10 轮后封顶 100（不溢出），小结卡下方提示「本节掌握度 已封顶 · 当前 100/100」。`masteryDelta <= 0` 但仍 completed 时至少加 8 分，保证学生**完成一题永远拿到正反馈**。
  - 跨小节独立：在 16.1 / 16.2 / 16.3 三节分别完成，首页能看到各自独立的「已完成 N 轮 · X/100」徽标。
  - 失败容错：手动清空 `shared_preferences` 后重开 App（或第一次安装），首页 16.3 自动回落到「可练习」，**不**显示 `null` / `--`，不影响课程目录展示；后端临时不可用也不影响进度展示（进度是纯本地的）。
- **预估耗时**：90 秒

## 8. 多章节 Mock 题切换
- **一句话描述**:同一讲题页根据章节展示不同的二次根式 Mock 题（16.1 取值范围 / 16.2 乘除 / 16.3 加减）。
- **演示要点**：返回首页 → 进入 16.1 看到「`\sqrt{2x-6}` 有意义时 $x$ 的取值范围」；进入 16.2 看到「`\sqrt{12} \cdot \sqrt{3}`」；每道题 Mock 对话都围绕该节核心知识点（被开方数非负 / `\sqrt{a}\cdot\sqrt{b}=\sqrt{ab}` 的前提 / 同类二次根式）。
- **预估耗时**：60 秒
