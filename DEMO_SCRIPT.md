# AI 费曼 · Demo 演示提纲

> 每完成一个可演示的功能或逻辑单元，立即在此追加条目（按 `AGENTS.md` 第 0.6 条要求）。
> 演示前请确认：Flutter 客户端已构建到 Android 设备，后端可选启动（首页右上角 API 标识会变绿）。

---

## 1. 课程目录全貌展示
- **一句话描述**：进入 App 首页，展示「人教版 · 初中数学」6 册 29 章 90 节完整目录，验证「壳子做全、肉先填一块」。
- **演示要点**：
  - 顶部 Hero 卡片指明「V1 仅开放八年级下册 · 第十六章 二次根式」，附带 16.1 / 16.2 / 16.3 三个绿色徽标。
  - 默认展开八年级下册，可练习章节以知性蓝高亮、带「可练习」标签；其他章节灰色 + 锁形图标 + 「即将上线」。
  - 点击未开放章节弹出温和提示，不进入练习页。
- **预估耗时**：30 秒

## 2. 二次根式讲题闭环（手写 → 多 Agent 追问）
- **一句话描述**：点击「16.3 二次根式的加减」进入讲题页，写出 `\sqrt{12}-\sqrt{27}` 的两行化简过程并提交，左侧小明 + 李老师按角色追问。
- **演示要点**:
  - 横屏平板呈现左 40% 多 Agent 对话区 + 右 60% 手写板的双栏布局。
  - 右侧手写板带横线辅助、`RepaintBoundary` 隔离重绘；支持「撤销」「清空」「提交讲解」。
  - 每停笔 1 秒以上自动切到下一 `step_id`，右上角实时显示「已识别 N 步」。
  - 点击「提交讲解」后约 0.6s 出现「正在让同学听讲…」呼吸气泡，紧接着出现小明的疑问 + 李老师的引导。
  - 气泡里的「看这一步」按钮可触发画布对应 `step_id` 的浅黄霓虹光晕 + 笔迹换色（湖青）效果。
  - AppBar 右上角徽标随轮次切换为「已完成 N 轮」。
- **预估耗时**：90 秒

## 3. 后端 Mock 多 Agent 追问（第二轮闭环）
- **一句话描述**：在 16.3 讲题页写两步思路 → 点击「提交讲解」→ Flutter 通过 `POST /lecture/submit` 真实请求 FastAPI，后端返回小明 + 李老师两条结构化追问。
- **演示要点**：
  - 演示前先启动后端：`cd main && uvicorn app.main:app --host 0.0.0.0 --port 8001 --reload`，首页右上角徽标变绿表示「API 已连接」。
  - 点击「提交讲解」后按钮进入 loading（旋转图标 + 「正在让同学听讲…」），按钮整体禁用避免重复点击。
  - 后端 ~50ms 返回 200 时，左侧依次出现「系统已收到第 N 轮讲解」「小明追问」「李老师引导」三条气泡，并按 `highlightStepIds` 自动让画布对应 `step_id` 亮起。
  - 关掉 uvicorn 后再点提交，气泡区底部弹出红色错误条「连不上后端（…）。请确认后端已启动…」+「重试」按钮，**手写内容不清空**，重新打开后端后点重试即可继续闭环。
  - 用 `curl -X POST http://127.0.0.1:8001/lecture/submit -H 'Content-Type: application/json' -d '{...}'` 现场展示固定 JSON 契约。
- **预估耗时**：90 秒

## 4. 真实 AI 多 Agent 追问（第三轮闭环 · Kimi K2.6 关思考）
- **一句话描述**：在 16.3 讲题页写两步 `\sqrt{12}=2\sqrt{3}`、`\sqrt{27}=3\sqrt{3}` → 点击「提交讲解」→ FastAPI 调 `kimi-k2.6`（`thinking.type=disabled`）让旗舰模型在一次对话里同时扮演小明/大雄/班长/李老师，~10-15 秒后给出 1-2 条结构化追问气泡，每条都精准命中学生写的那两个 `step_id`。
- **演示要点**：
  - 演示前确认根目录 `.env` 已配置 `KIMI_API_KEY`，并启动后端 `cd main && uvicorn app.main:app --host 0.0.0.0 --port 8001 --reload`。
  - 按钮文案从「提交讲解」切到「AI 同伴思考中…」，左侧呼吸气泡显示「AI 同伴正在阅读你的步骤、写追问……」，**和第二轮 0.5s 假 loading 明显不同**，让评委直观感到「这次真的在调 LLM」。
  - 后端 ~10-15s 返回后，按角色铺出 1-2 条气泡。**重点是质量而不是速度**：
    - 16.3 实测：小明会精确指出「`\sqrt{12}=2\sqrt{3}` 里为什么开出来是 2 不是 `\sqrt{2}`」，班长引导「合并系数 2 减 3」但不直接给答案 `-\sqrt{3}`。
    - 16.1 实测：班长会要求把规则浓缩成「定义域口诀」，老师会拓展追问「若题目改成 `\sqrt{-x}`，口诀还能直接用吗？条件会变成什么？」。
    - 16.2 实测：小明追问 `\sqrt{a}\cdot\sqrt{b}=\sqrt{ab}` 的前提条件，班长精确给出 `a\ge 0, b\ge 0` 并直接引用 `step_x` 名要求把条件补到那一步前面。
  - 每条气泡都会自动让画布对应 `step_id` 亮起（霓虹光晕 + 笔迹换湖青）。
  - 现场可在终端 `tail -f main/logs/app_*.log`，日志里 `source=llm turns=2` 即代表走的是真实 LLM；`source=fallback` 即走第二轮 Mock 兜底。
  - 用 `curl` 把 `KIMI_API_KEY` 临时清掉（`KIMI_API_KEY="" uvicorn ...`）重启后端，再点提交，气泡照常出现 ——**Demo 链路永不中断**，说明 fallback 已就绪。
  - 若 LLM 返回非 JSON / 字段非法 / 引用了不存在的 `step_99`，后端校验层会自动剔除或回退首步 `stepId`，前端不会出现"点不亮"或"乱码"。
- **预估耗时**：120 秒

## 5. 多章节 Mock 题切换
- **一句话描述**:同一讲题页根据章节展示不同的二次根式 Mock 题（16.1 取值范围 / 16.2 乘除 / 16.3 加减）。
- **演示要点**：返回首页 → 进入 16.1 看到「`\sqrt{2x-6}` 有意义时 $x$ 的取值范围」；进入 16.2 看到「`\sqrt{12} \cdot \sqrt{3}`」；每道题 Mock 对话都围绕该节核心知识点（被开方数非负 / `\sqrt{a}\cdot\sqrt{b}=\sqrt{ab}` 的前提 / 同类二次根式）。
- **预估耗时**：60 秒
