# AI Code Agent 执行指令：第七轮本地小题库、难度标签与下一题轮换闭环

> 本页用于约束 AI Code Agent 的第七轮实现。开始写代码前，必须先阅读 `AGENTS.md`、`项目规划/planV1.md`、`MOBILE_STYLE.md`、`docs/AI_CODE_AGENT_BRIEF.md`、`docs/AI_CODE_AGENT_BRIEF_ROUND2.md`、`docs/AI_CODE_AGENT_BRIEF_ROUND3.md`、`docs/AI_CODE_AGENT_BRIEF_ROUND4.md`、`docs/AI_CODE_AGENT_BRIEF_ROUND5.md`、`docs/AI_CODE_AGENT_BRIEF_ROUND6.md`，并以本页作为本次任务边界。

## 1. 当前进度判断

前六轮已经完成：

- Flutter 学生端首页与二次根式讲题页骨架。
- 手写板、多 Agent 追问、真实 LLM、fallback。
- 学生口述 / 步骤说明 / LaTeX 手动输入。
- 本地多轮追问上下文。
- 完成一题后的本地掌握度、完成次数与首页进度展示。

当前主要短板：每个 V1 小节目前只有一道固定 Mock 题。学生点「下一题」后，页面只是清空状态再做同一道题，不像真实练习产品。

## 2. 本轮目标

做出一个更像真实练习产品的本地小题库闭环：

**每个 V1 小节内置 3 道题 → 首页显示题量 → 讲题页显示题号/难度/标签 → 点「下一题」切换到同小节下一道题 → 完成记录仍累计到该小节掌握度。**

本轮不接后端题库，不做 LLM 自动出题，只把 V1 二次根式章节的本地练习体验打通。

## 3. 严格范围

本轮只做以下事情：

- 把 `MockLectureRepository` 从“每节 1 题”升级为“每节多题”。
- 16.1 / 16.2 / 16.3 每节至少 3 道题。
- 每道题带轻量难度信息：基础 / 巩固 / 挑战。
- 每道题带 1-3 个知识标签，用于讲题页展示。
- 讲题页持有当前题目索引。
- 页面显示题号、难度、标签，例如「第 2 / 3 题 · 巩固 · 化简 / 同类二次根式」。
- 点击「下一题」进入同小节下一道题，并清空当前题临时状态：
  - 手写板
  - 高亮
  - 学生口述
  - 步骤说明 / LaTeX
  - 多轮 `history`
  - 本题讨论气泡
- 小节内题目做循环：第 3 题点下一题回到第 1 题。
- 首页可练习小节显示本节题量，例如「3 道题 · 可练习」。
- 完成任意一道题后，仍按第六轮逻辑累加该 `sectionId` 的本地掌握度。

## 4. 本轮不做

以下能力继续禁止展开：

- 不做后端题库接口。
- 不做 LLM 自动出题。
- 不做拍照识题。
- 不做难度自适应算法。
- 不做数据库持久化。
- 不做账号级错题本。
- 不做题级完成记录。
- 不做题目收藏、错题标记、题目搜索。
- 不做家长端、排行榜、晶石。
- 不改 `/lecture/submit` 请求/响应契约，除非只是传入不同的 `questionId` / `questionPrompt`。

## 5. 题库设计

建议继续使用本地 Dart Mock，不引入 JSON 题库文件，保持第七轮轻量。

### 5.1 数据模型

建议把 `LectureQuestion` 轻量加厚：

```dart
final int difficulty; // 1=基础, 2=巩固, 3=挑战
final List<String> tags; // 如 ["取值范围", "非负条件"]
```

UI 中不要直接显示 `difficulty=1` 这种开发字段，要转成中文：

- `1` → `基础`
- `2` → `巩固`
- `3` → `挑战`

标签只用于前端展示，不传后端；后端只需要 `questionId` / `questionPrompt`。

### 5.2 推荐题目

16.1 二次根式概念与取值范围：

- `q-s16-1-001`：判断 `\sqrt{2x-6}` 有意义时 `x` 的取值范围。难度：基础；标签：取值范围、非负条件。
- `q-s16-1-002`：判断 `\sqrt{5-x}` 有意义时 `x` 的取值范围。难度：巩固；标签：取值范围、移项。
- `q-s16-1-003`：判断 `\sqrt{x+3} + \sqrt{2-x}` 有意义时 `x` 的取值范围。难度：挑战；标签：公共定义域、不等式组。

16.2 二次根式乘除：

- `q-s16-2-001`：化简 `\sqrt{12} \cdot \sqrt{3}`，说明乘法法则前提。难度：基础；标签：乘法法则、前提条件。
- `q-s16-2-002`：化简 `\sqrt{50} \div \sqrt{2}`。难度：巩固；标签：除法法则、化简。
- `q-s16-2-003`：化简 `\sqrt{8} \cdot \sqrt{18}`。难度：挑战；标签：乘法、完全平方数。

16.3 二次根式加减：

- `q-s16-3-001`：化简 `\sqrt{12} - \sqrt{27}`。难度：基础；标签：最简二次根式、同类二次根式。
- `q-s16-3-002`：化简 `2\sqrt{8} + \sqrt{18}`。难度：巩固；标签：化简、合并同类项。
- `q-s16-3-003`：化简 `\sqrt{45} - 2\sqrt{20} + \sqrt{5}`。难度：挑战；标签：多项合并、负号。

每道题都要有：

- `questionId`
- `sectionId`
- `sectionLabel`
- `prompt`
- `hint`
- `referenceSteps`
- `difficulty`
- `tags`

## 6. Repository 要求

建议把 `MockLectureRepository` 改成以下能力：

```dart
List<LectureQuestion> questionsForSection(String sectionId);
LectureQuestion questionForSection(String sectionId, {int index = 0});
int questionCountForSection(String sectionId);
String difficultyLabel(int difficulty);
```

约束：

- 旧调用 `questionForSection(sectionId)` 仍可工作，默认返回第 1 题。
- 未知 `sectionId` 回退到 16.1 第 1 题。
- `index` 超出范围时用 modulo 循环，而不是抛异常。
- `questionCountForSection` 供首页展示「3 道题」。
- `difficultyLabel` 或同等 helper 供 UI 使用，避免页面里到处写 switch。

## 7. 讲题页交互要求

### 7.1 题号展示

在讲题页题面卡片或 AppBar 中显示：

- `16.3 二次根式的加减 · 第 1 / 3 题 · 基础`

题号变化时，题面、提示、参考步骤、难度标签、知识标签都要随之更新。

### 7.2 题目卡片信息

题面卡片中增加一行轻量信息：

- 难度 chip：`基础` / `巩固` / `挑战`
- 标签 chip：例如 `最简二次根式`、`同类二次根式`

要求：

- chip 风格遵守 `MOBILE_STYLE.md`，小而清楚，不要做成游戏化徽章。
- 标签只展示 1-3 个，避免题面卡片变拥挤。
- 手机竖屏下标签可换行，但不能挤压手写板主区域。

### 7.3 下一题行为

点击「下一题」时：

- 题目索引 +1，超过题数后回到 0。
- 清空手写板、输入区、步骤说明、LaTeX、history、turns、高亮、错误状态。
- 重置 `_round`。
- 重新显示老师 intro 气泡，气泡里应引用新题的 `sectionLabel` 或题面。
- 不清空本地进度。

### 7.4 再讲一遍行为

点击「再讲一遍」时：

- 保持当前题目索引不变。
- 清空当前题临时状态。
- 不清空本地进度。

### 7.5 完成态

完成题目后，完成卡片继续展示本题总结与掌握度变化。

如果学生点「下一题」，新题开始后完成卡片消失。

## 8. 后端要求

后端不需要新增接口。

但要确认：

- `/lecture/submit` 能接收新的 `questionId`。
- `questionPrompt` 会进入 LLM Prompt。
- LLM 追问必须围绕当前题面，不要因为 section 相同就固定追问旧题。
- fallback 可以继续按 section 兜底，不要求每道题都有专属 fallback。

## 9. 首页要求

首页进度仍按小节展示，不按题目展示，但可练习小节要显示题量：

- 未完成：`3 道题 · 可练习`
- 已完成：`已完成 2 轮 · 掌握度 20/100`

如果空间允许，可在小节副标题或小号文字里显示：

- `本节内置 3 道讲题练习`

本轮不需要展示“小节内第几题已做过”，避免引入题级历史。未上线章节仍只显示「即将上线」，不要显示题量。

## 10. 验收标准

本轮完成后必须满足：

- 16.1 / 16.2 / 16.3 每节至少有 3 道本地题。
- 进入任一可练习小节，讲题页显示「第 1 / 3 题」。
- 讲题页题面卡显示当前题的难度 chip 和知识标签 chip。
- 首页可练习小节在未完成时显示「3 道题 · 可练习」或同等题量信息。
- 点「下一题」后题面确实变化，不是同一道题重置。
- 第 3 题点「下一题」会回到第 1 题。
- 「再讲一遍」保持当前题不变。
- 提交不同题时，请求体里的 `questionId` 和 `questionPrompt` 随题变化。
- 完成任意题后，小节掌握度继续累计。
- 未上线章节不显示题量、不显示难度标签。
- 第六轮本地进度、第五轮多轮追问、第三轮 LLM fallback 不被破坏。

## 11. 建议测试

前端：

```bash
cd main/mobile
flutter analyze
flutter run
```

手动验证：

1. 从首页进入 16.3。
2. 确认显示第 1 / 3 题。
3. 确认题面卡显示难度 chip 和标签 chip。
4. 点「下一题」，确认切到第 2 / 3 题，题面、难度、标签同步变化。
5. 再点两次，确认第 3 / 3 后循环回第 1 / 3。
6. 在第 2 题完成一轮，确认首页 16.3 掌握度增加。
7. 返回首页，确认未完成小节显示「3 道题 · 可练习」。
8. 回到 16.1 / 16.2，确认也各有 3 道题。

如果无法真机运行，至少通过 `MockLectureRepository` 的简单断言或临时日志确认题目轮换。
