# Flutter 客户端风格规范（MOBILE_STYLE.md）

> **核心定位**：专注、温和、可信赖的初中数学学习空间（平板优先）。
> **优先级**：本文件高于一切临时审美判断。任何新建、重构 Flutter 页面/组件/样式/动画/交互前，**必须先读完本文件**。

---

## 1. 核心美学

**一句话定调**：
> 像光线充足、布置讲究的现代自习室——让学生愿意开口讲题，而不是面对冷冰冰的聊天框。

**视觉关键词**：
- **追求**：清晰层级、大触控区、柔和圆角、温暖适度留白、知性蓝/青主色、出色的中文与公式可读性。
- **远离**：紫蓝高饱和度渐变 Hero、赛博硬核风、过小按钮、密不透风的文字墙、无状态的空白加载。

## 2. 平台、布局与适配

| 项 | 平板端规范（横屏优先，10-12寸为主） | 手机端规范（竖屏适配） |
|:---|:---|:---|
| **主界面形态** | **全屏白板优先**：手写板占满工作区；题面为顶部窄条可点开；右侧纵向四个 **SVG 人设头像**（小明/大雄/班长/李老师，外圈表听懂状态）；工具一律 48dp 圆形图标 | **同左**：白板仍占主区域，头像轨与圆钮贴边浮层，禁止大块矩形讨论区/音频面板 |
| **最小触控区** | `>= 48×48 dp` (图标建议 `24dp`，Padding 至少 `12dp`) | `>= 48×48 dp` |
| **圆角 (Radius)**| 卡片 `16dp`，大组件 `24dp`，主按钮 `12dp`，小微组件 `8dp` | 卡片 `12dp`，按钮 `8dp` |
| **页面内边距** | `24dp` 边缘间距，模块间距 `16dp` | `16dp` 边缘间距，模块间距 `12dp` |
| **文字排版规范**| 正文字号 `16sp`，**行高 (height) 必须限制在 1.4 ~ 1.6 之间**。*（注：初中数学涉及根式、分式，低行高极易导致排版挤压叠字）* | 正文字号 `15sp`，行高 `1.3 ~ 1.5` |

## 3. 色彩与字重（Material 3 调色板）

本项目采用温和知性的“自习室”色系，严禁使用炫酷暗黑、荧光等电竞色彩。

| 角色 | 推荐 Hex 色值 | 视觉用途 |
|:---|:---|:---|
| **Primary** | `#1E40AF` (知性深蓝) / `#0891B2` (智慧湖青) | 页面主操作按钮、已选中状态、进度高亮 |
| **Secondary** | `#3B82F6` (清朗浅蓝) | 次要按钮、状态标签（例如“可练习”） |
| **Background** | `#FBFBFA` (温暖暖白/纸张色) | 全局背景。代替刺眼纯白，降低学生长时间紧盯屏幕的视觉疲劳 |
| **Surface** | `#FFFFFF` (白) | 卡片、对话框背景，需带淡灰色投影或 1dp `#ECEFF1` 描边 |
| **Outline** | `#E2E8F0` (柔和灰) | 边框线、分割线、未上线章节置灰边框 |
| **Coming Soon** | `#94A3B8` (中灰) + `opacity: 0.5` | 置灰、锁定状态组件 |
| **Error** | `#EF4444` (警示红) | API 报错提示、录音权限失败、网络重试提醒 |

### 暗色模式
V1 阶段**仅支持亮色模式**（防止多色渲染带来排版和公式不一致的视觉坑点）。若后续扩展暗色，背景色必须控制在柔和暗灰（如 `#121212` 或 `#1A1A1E`），**严禁直接使用 `#000000` 纯黑**。

## 4. 专项组件与核心交互规范

### 4.1 数学公式渲染（LaTeX）
在 AI 费曼中，公式（特别是二次根式）是核心教学内容。公式渲染质量直接决定了 App 的成败。

1. **渲染库选型约束**：
   - **禁止使用基于 WebView 的公式渲染器**（如 `flutter_tex`），其不仅会在低端平板上导致显著的滑动卡顿，还可能由于内存泄露引发 OOM 崩溃。
   - **强制采用基于 Flutter 原生 Canvas 渲染的方案**（如 `flutter_math_fork` 渲染 KaTeX 或使用原生 Markdown 配合 KaTeX 解析插件）。
2. **格式预处理**：
   - 客户端接口层收到 LLM 的流式输出后，需在渲染前进行正则清洗：将 `\(...\)` 转换为 `$ ... $`（行内），将 `\[...\]` 转换为 `$$ ... $$`（块级）。
3. **视觉排版**：
   - **行内公式**：字号必须与上下文文字完全一致，底部对齐，防止公式过大导致行高异常撑开。
   - **块级公式**：自动水平居中，并在上下提供 `8dp` 的垂直间距，公式字体颜色跟随正文主色。

### 4.2 手写板（Ink Canvas）规范
手写轨迹是学生“向 AI 讲题”时的核心交互。

1. **笔迹渲染与样式**：
   - 默认画笔颜色使用 **Secondary** 色（`#3B82F6`），笔宽 `3.0 dp`，笔头采用 `StrokeCap.round`（圆润），确保书写柔和。
   - 书写轨迹采集需做防抖（双线性插值或贝塞尔曲线优化），防止出现折线锯齿。
2. **重绘性能保障 (RepaintBoundary) [重要]**：
   - 运行手写板的 Widget 必须被包裹在 `RepaintBoundary` 中，与左侧的 SSE 流式文本区做**重绘级别隔离**。
   - 保证流式文本逐字吐出时，手写板帧率稳定维持在 `60fps ~ 120fps`，绝对不允许打字机效果频繁重绘导致手写断笔、粘滞。
3. **手写板工具栏**：
   - **清除 (Clear)**、**撤销一步 (Undo)** 必须以高辨识度的大图标（带有 `padding` >= `12dp`）放置于手写板右上角或右下角。
   - 提供「讲完了/一键 OCR 识别」的大型主操作按钮，引导学生完成“手写 -> 上传识别 -> Agent 追问”的闭环。
   - **防误触 (Palm Rejection)**：在平板上默认屏蔽多点触控（只识别第一根手指或触控笔的轨迹），避免学生手掌贴在平板边缘时发生笔画乱飞。

### 4.3 实时双工语音交互（Real-time Duplex Voice Interaction）
在 AI 费曼中，语音交互不采用传统对讲机模式（录音-上传-等待-播放），而是采用**流式双工（Continuous Streaming & Real-time Duplex）交互模式**。这种模式下，麦克风保持开启，ASR 实时流式识别，对话实时更新，支持智能打断。

1. **取消单次录音时长限制**：
   - 数学讲题和思维推导过程往往长达 2 至 5 分钟，甚至更长。**取消 60 秒硬性时长限制**。
   - 系统支持持续流式会话，直到学生主动点击挂断，或系统触发超长不活动安全保护（如 10 分钟无声）。

2. **流式 ASR 上屏预览 (Real-time Transcription Preview)**：
   - 客户端麦克风采集音频分片 (PCM Chunks)，通过流式长连接 (WebSockets / HTTP Chunked) 实时送往后端。
   - **流式字迹反馈**：学生一边说话，识别出的文字必须**在 200ms 内以淡半透明状态 (opacity: 0.6) 实时递增呈现在屏幕聊天框中**。这种流式上屏使用户能随时看到 AI 听懂了什么，消除“AI 到底有没有在听”的焦虑。

3. **自适应静音检测与礼貌气口停顿 (Adaptive VAD & Conversational Pause)**：
   - **数学思维需要停顿**：传统的 VAD (语音活动检测) 往往在用户停顿 500ms~800ms 时就会切断并触发回复。这会导致学生在“看步骤、想下一步”时被强行打断。
   - **智能断句与礼貌契机**：系统遵循「AI 绅士礼貌原则」，绝不在学生发声或书写途中粗暴插话。即使学生在步骤中出现致命错误，AI 也要静静观望，等学生讲完当前的句子。
   - **逻辑气口检测**：当前端检测到学生完成表达，并产生 **1.5 秒以上的自然静音停顿（逻辑气口）**时，后台状态机和决策引擎才允许触发讨论或切入。
   - 提供「思考中」挂起按钮：学生可一键使 ASR 暂停（Pause Listening），算完后再点击继续。

4. **智能打断与回消 (Barge-in & Echo Cancellation)**：
   - **学生打断 AI (主权在学生，社交化过渡)**：当 AI 正在播放 TTS 语音、而学生突然灵光一现开口说话或纠错时，为了防止环境噪音、叹气或短暂的“呃”产生误打断，引入 **300ms 连续人声检测防抖阈值**。
   - **音频渐隐与缓冲清空**：一旦触发打断，客户端在 **200ms 内执行音频渐隐（Audio Fade-out）**，平滑停止 TTS 播放并清空音频缓冲区。
   - **打断的社交包装**：AI 被打断时，对应的虚拟角色气泡立即展示贴心的鼓励提示，例如小明气泡显示“*（啊，原来你懂了！那我听你讲）*”，李老师气泡显示“*（你有新想法啦，快说说看）*”，无缝切回倾听状态。
   - **回声消除 (AEC)**：在真机平板扬声器播放 TTS 时，由于麦克风同时开启，必须开启 Native 级别的回声消除，防止 AI 听到并识别它自己播放出来的声音。如果无法通过软件完美消除，界面需温和提示“戴上耳机体验更佳”。

5. **双工交互状态机与视觉动效反馈**：
   - **`LISTENING` (听取与流式识别)**：麦克风激活。按钮外围展示随着音量/分贝大小实时律动的**涟漪或波形动效 (Audio Waveform)**。显示半透明实时识别词流。
   - **`THINKING` (AI 思考/追问中)**：VAD 判定学生话毕。波形转为温和的呼吸律动（Pulse），左侧显示打字机流式出字。
   - **`SPEAKING` (AI 语音输出)**：TTS 播放。大按钮显示温和的同频振幅，同时保持对学生声音的监听，随时响应“智能打断”。

### 4.4 流式打字机效果（SSE 渲染）
1. **渲染更新控制**：
   - 使用局部 State 管理流式增量段落（Delta），避免调用 `setState` 刷新整个讲题页面。
   - 对话流列表在接收到新文本后，应在后台滑动监听器中执行平滑平移（`animateTo`，曲线 `Curves.easeOut`，耗时 `100ms`），使用户视线始终跟随打字机的底部。
   - 流式过程中的 Sentinel 标签（如 `[THINKING]`、`[DONE]`）必须通过正则在 UI 渲染层彻底过滤，严禁展示给学生。

## 5. 项目目录结构约定

保持清晰的 Flutter 工程架构，防止大型组件混杂。

```
main/mobile/lib/
├── main.dart
├── config/              # 全局配置、API BaseURL 变量、全局常量
├── data/                # Curriculum 数据实体、Repository、本地缓存
├── services/            # 网络请求 (Dio/SSE)、语音录制、ASR、TTS、持久化
├── theme/               # ThemeData、调色板（Hex定义）、Typography 排版、圆角定义
├── widgets/             # 可复用、解耦的通用组件
│   ├── formula_text.dart# 原生公式渲染封装
│   ├── hand_canvas.dart # 带有 RepaintBoundary 的手写板
│   └── audio_wave.dart  # 录音波形动画组件
└── pages/               # 独立页面目录
    ├── home_page.dart   # 课程章节大纲主页
    └── lecture_page.dart# 讲题、手写双栏交互核心页
```

## 6. 与后端协作规范

1. **统一 API 配置**：
   - 所有的 Base URL 及路由 endpoint 统一维护在 `lib/config/api_config.dart`，**绝对不允许**在业务代码、Service 层硬编码。
2. **真机与局域网调试**：
   - 在开发及真机测试时，将 Base URL 配置为开发机局域网 IP（例如 `http://192.168.1.100:8001`），禁止配置 `localhost`。
3. **鉴权机制**：
   - Token 走 `Authorization: Bearer <token>` 统一注入网络拦截器（Dio Interceptors），避免因 Session 跨端或 Cookie 不匹配引起 401 报错。
4. **SSE 流解析容错**：
   - 由于网络切片，必须在 `Stream` 监听层配置一个 Buffer（缓冲区），确保能把分片的 Chunk 拼全后再抛给 JSON Parser（格式参考：`json.loads(line.replace("data: ", ""))`），防止断断续续的 JSON 截断错误。
